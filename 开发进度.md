## 已完成能力
- 统一响应结构：`{ code, message, data }`
- JWT access token 鉴权链路可用（Bearer Token）
- refresh token 已接入简化方案（MVP）
- User / EmailVerificationCode / WalletChallenge / UserWallet / TwitterAccount / AutomationConfig / ActivityLog 等模型已纳入 AutoMigrate（另含 Post / Agent / Task 表结构）
- GORM AutoMigrate 与 `backend/internal/database/migrate.go` 一致
- Wallet 绑定闭环可跑通（challenge -> bind -> me 可见地址 -> unbind）
- 钱包绑定已接入 EVM 签名恢复校验，且禁止跨用户绑定同一钱包
- Accounts 页面已接入真实 OAuth 绑定/回调/删除链路
- Dashboard 已接入真实 overview 数据并支持 loading/error/retry
- Accounts 与 Dashboard 已通过 refresh bridge 联动，绑定/解绑后可自动刷新连接数
- **Agents 页面**：使用 **`/automations*`** 真实配置，不是 `GET /agents` 占位接口
- **Activity / Billing**：列表与订阅类接口已接；活动表首次空库会插入演示行（EnsureDefaults）
- **X OAuth**：YAML 配置 `x_oauth`；回调与 token 交换带结构化日志；前端授权可 **弹窗** 打开，回调经 `postMessage` 通知父页
- **头像**：`next.config` 放行 `pbs.twimg.com` / `api.dicebear.com`；Account 卡头像 `next/image` 使用 `unoptimized` 避免本地无法访问图床时裂图

## 当前已知限制（MVP）
- refresh token 为简化实现，未做服务端会话撤销/多端管理
- **`GET /automations/runtime-status`**：`queue_depth`、`last_success_at`、`retries_last_24h` 等为**占位算法**（真实统计待任务/队列接入）
- **Billing**：支付方式列表常为空；订阅侧为试用+MVP 逻辑，非完整计费中台
- **Activity**：新用户首次拉列表可能看到 EnsureDefaults 写入的演示数据
- **后端访问 X**：`api.x.com` / `pbs.twimg.com` 需网络可达；否则换 token 超时或头像裂图（已通过 `unoptimized` 等缓解头像）
- `GET /posts`、`GET /agents`：仍为占位；Accounts「管理/重连」部分交互仍为占位或 console.log，待产品接口
