# Support

For reproducible bugs, documentation fixes, and focused feature proposals, use
GitHub Issues in this repository.

Before opening an issue:

- Search existing issues first.
- Include clear reproduction steps for bugs.
- Include screenshots for UI or website problems.
- Remove secrets, tokens, private endpoints, customer data, and logs with
  sensitive values.

For security vulnerabilities, do not open a public issue. Follow
[SECURITY.md](SECURITY.md) and report privately through GitHub Security
Advisories.

This public repository does not provide production hosting, server operations,
or private deployment support.
